Tier 1 NPU functional units explained: 2D strided DMA, activation LUT, and per-channel requantization

2D strided DMA + zero-padding insertion
the smart memory copier
Your feature map lives in DRAM as flat rows back-to-back. When you need a tile (a small rectangle from it), its rows are not next to each other — there is a gap between each tile row equal to the columns you do not need. Without this unit, your CPU has to manually copy row-by-row, burning cycles. The DMA does the whole thing automatically from one descriptor: base address, row stride, tile size — done.

At the image edge, a tile hangs off the side of the image. The kernel would read out-of-bounds memory. Instead the DMA inserts synthetic zero cells on the border, so the systolic array always sees a full properly-padded tile and never needs to know it is at the edge.

One descriptor: base ptr + row stride + tile dims + pad counts. The systolic array never knows it is at the image edge.
Without this, the ARM has to manually re-pack every tile before each layer dispatch — often the single biggest source of latency in naive NPU implementations.
Activation LUT
the answer key
SiLU is x · σ(x), where σ(x) = 1 / (1 + e^−x). Computing that from scratch in hardware means an exponent, an addition, a division, and a multiply — expensive to pipeline, tricky to keep precise in fixed-point. But INT8 only has exactly 256 possible input values (-128 to 127). So you pre-compute all 256 answers offline and store them in a 256-byte table baked into a block RAM in the VPU pipeline. At runtime: input arrives, it is used as an array index, the answer comes straight out the other side — one clock cycle, zero math. And if you load different bytes into the table before a layer runs, you get a completely different activation function. ReLU, GELU, Hardswish — zero RTL changes required.

Per-channel requantization pipeline
the custom squeezer
The systolic array accumulates into INT32 — numbers up to ±2 billion. The next layer and your SRAM want INT8 — numbers between −128 and 127. You need to scale down and clip. The catch: each output channel needs its own scale factor, determined during quantization. Channel 0's weights might be 3× bigger than channel 1's, so their raw INT32 outputs live on completely different scales. One global divisor will crush some channels into ±127 (saturated — all detail lost) while rounding others to zero (underflow — also dead).

Hardware: a SRAM-resident coefficient buffer holding (M, S) per output channel. The VPU loads the next pair each cycle as it walks the output tile. Skipping per-channel scaling is the most common first cause of accuracy collapse in first-draft NPU implementations.
